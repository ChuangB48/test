#  壹、上機演練試題
一、 本次試題主要的目標是讓考生了解在 Linux 平台上進行 micro-ROS 應用程式
開發流程。
<br>
二、 本份文件摘要與此目標相關重要的概念,實際的執行步驟,請依照 micro-ROS
的引導文件 [First micro-ROS Application on Linux](https://micro.ros.org/docs/tutorials/core/first_application_linux/) 操作。注意:進行 micro-
ROS 程式開發,電腦需要預先安裝 Ubuntu Linux 22.04 以及 ROS 2 Humble 軟
體,詳見上述文件。
<br>
三、 本試題演練主要是運行一個基於 micro-ROS 環境之 Ping-Pong 應用程式,此程
式由兩個節點(Node1、Nodes2)組成,並透過下圖的模式進行資料的交換,具
體資料流與相關 Message Topics 資訊請見下圖。需要注意的是 micro-ROS 系統
與 ROS 2 系統一樣,主要是透過 Publish-Subscribe 模式進行溝通,為了完成
Ping 與 Pong 的資料傳遞動作,每個節點同時具備 Publisher 與 Subscriber 的角
色,並同時使用相同訊息型別(std_msgs/Header)進行資料傳遞。例如:Node1
透過/microROS/ping 的 Message Topic 將資料傳給 Node2,而實際的訊息型別則
為 std_msgs/Header。Node1 將資料傳給 Node2 稱之為 Ping,而 Node2 將資料透
過/microROS/pong 的 Message Topic 傳給 Node1 則稱之為 Pong。
<br>
四、 本試題演練之 micro-ROS 應用程式在進行資料溝通時,實際上會透過 ROS 2
系統的協助來傳遞資料。下圖為 micro-ROS 應用程式開發時使用的軟體堆
疊,以及 micro-ROS 與 ROS 2 之間的溝通方式(透過 ROS 2 Agent 來協助資料
傳遞)。ROS 2 Agent 可讓資源有限的嵌入式設備與 主機端(Host)標準 ROS 2 系
統進行通訊。值得注意的是 micro-ros-agent 在圖左側之主機端執行,由於 Host
使用標準 DDS 協定,而嵌入式平台採用的是輕量級的 Micro XRCE-DDS 協
定,兩者無法直接通訊,micro-ros-agent 負責將兩種協定之間的訊息進行轉
換,因此嵌入式平台之 micro-ROS 的節點便能與其他 ROS 2 節點進行資料交
換,以此實現低階電腦系統與一般電腦系統之資料通訊整合。
試問:
<br>
1. micro-ROS 與 ROS 2 之間的關係為何? 在什麼樣的設備上適合使用
ROS 2,在什麼樣的設備上適合使用 micro-ROS?
<br>
3. 如果要透過 C / C++/ Python 實作基於 ROS 2 的資料傳輸,達到 Host
端與 嵌入式設備端能夠收發資料,應當如何實作?
<br>
5. 資料傳送時,可以傳送的資料型態有什麼限制?是否可以傳送客製
化的資料型態(訊息型別)?
<br>
7. 上述的試題演練是在 Linux 環境上進行 micro-ROS 的開發以及模擬
執行,假如需要在實體嵌入式設備上運行 micro-ROS 系統與應用,
整體開發流程會有甚麼不同?
